# CLAUDE.md — PilotGear EG Store + Admin

Project memory for the PilotGear EG e-commerce site and Shopify-style admin.
Read this first before working on this project.

## What this is
A complete storefront **and** admin dashboard for **PilotGear EG**, an Egyptian brand
selling aviation-themed accessories (mainly keychains) plus a desk lamp and coasters.
Founded to bring aviation passion to Egypt for pilots, aspiring pilots and gift buyers.

- **Stack:** React 18 + TypeScript + Vite + Tailwind CSS + React Router 6 + Recharts + lucide-react
- **Data:** 100% self-contained in the browser via `localStorage` (no server/database).
- **Payments:** Cash on Delivery only (Egypt).
- **Deploy target:** Cloudflare Pages (build `npm run build`, output `dist`, `public/_redirects` handles SPA routing).

## Run / build
```bash
cd pilotgear-store
npm install
npm run dev      # http://localhost:5173
npm run build    # type-checks + builds to /dist
```
Preview tooling: `.claude/launch.json` defines server name `pilotgear` (runs dev with `--prefix pilotgear-store`).

## IMPORTANT: data versioning
Store data persists in `localStorage` under `KEY` in `store.ts`. Because a returning
user's browser already holds the old seed, **any change to `seed.ts` only takes effect
if you bump the version**: increment both `KEY` and `CART_KEY` in `store.ts`
(e.g. `pilotgear:data:v3` → `v4`) and the `version` field in `buildSeed()`.
Current version: **v3**.
Alternatively the user can use **Admin → Settings → Factory reset**.

## Product images
All product images were standardised through a Python/PIL pipeline onto one uniform
background (`#F6F8FA`): composite transparent PNG → recolour near-white to bg → trim to
content → centre on a square canvas (900×900). Output as `public/products/<slug>/1.png`, `2.png`, …
- The pilot keychain source photos had a red **"REMOVE BEFORE FLIGHT" corner watermark** — removed by using the clean `dqvcfiir` photo and erasing the corner on others.
- To regenerate, re-run the PIL pipeline against `../Product images/` (prefer `No BG`/`Photoroom`/`No WM` transparent sources).

## Known constraints / gotchas
- Mobile nav drawer must be rendered **outside** `<header>` — the header uses
  `backdrop-blur`, and a `backdrop-filter` ancestor re-anchors `position:fixed` children,
  which previously broke the full-screen drawer overlay. Keep it a sibling of `<header>`.
- Variant image swapping relies on `variant.image`; A330 variants intentionally have no
  per-colour image (no clean source), so the selector is functional only.
- Because data is per-browser, customer orders are stored in the customer's browser, not
  centrally. To sync orders across devices, swap the `store.ts` data layer for a real API.
